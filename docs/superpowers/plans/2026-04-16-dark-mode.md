# Dark / Light Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a system-aware dark/light mode toggle to the Eat This Berlin app, accessible via the burger menu and site footer, persisted in localStorage.

**Architecture:** A `[data-theme="dark"]` attribute on `<html>` drives all dark styles via CSS overrides. A no-flash inline `<script>` in `<head>` sets the attribute before paint. `initTheme()` in `app.js` wires the iOS-style toggle in both burger drawer and footer template, keeping them in sync.

**Tech Stack:** Vanilla CSS custom properties, vanilla JS, localStorage, `prefers-color-scheme` media query.

## Task 1 — No-flash init script in `<head>`

**Files:**
- Modify: `index.html` — insert 8 lines after `<html lang="en">`

The script must run before any CSS renders to prevent a white flash on dark-preference devices.

- [ ] **Step 1: Open `index.html`, find line 2 (`<html lang="en">`) and insert the script block**

The `<head>` currently opens at line 3. Add this as the **first child of `<head>`**, before the charset meta tag:

```html
<script>
  (function () {
    var s = localStorage.getItem('theme');
    var dark = s === 'dark' || (!s && window.matchMedia('(prefers-color-scheme: dark)').matches);
    if (dark) document.documentElement.setAttribute('data-theme', 'dark');
  }());
</script>
```

Insert it so the `<head>` block looks like:

## Task 7 — JS: `initTheme()` function

**Files:**
- Modify: `js/app.js` — insert `initTheme()` call near the top of the `DOMContentLoaded` handler (after `bodyOverflow` setup, around line 100)

- [ ] **Step 1: Add `initTheme()` to `app.js`**

Find the line `document.addEventListener('DOMContentLoaded', () => {` (line 86). Insert the full `initTheme` function and its call **after** the `bodyOverflow` block (around line 98, after `window.bodyOverflow = bodyOverflow;`):

```javascript
// ============================================
// THEME — Dark / Light mode
// ============================================
function initTheme() {
  const root = document.documentElement;

  function applyTheme(dark, persist) {
    if (dark) {
      root.setAttribute('data-theme', 'dark');
    } else {
      root.setAttribute('data-theme', 'light');
    }
    if (persist) {
      localStorage.setItem('theme', dark ? 'dark' : 'light');
    }
    // Keep both toggles in sync
    syncToggles(dark);
  }

  function syncToggles(dark) {
    // Called every time theme changes — no-op if toggles not yet in DOM
    ['themeToggleBurger', 'themeToggleFooter'].forEach(function (id) {
      var el = document.getElementById(id);
      if (el) el.setAttribute('aria-pressed', dark ? 'true' : 'false');
    });
  }

  function isDark() {
    return root.getAttribute('data-theme') === 'dark';
  }

  function wireToggle(id) {
    var el = document.getElementById(id);
    if (!el) return;
    el.addEventListener('click', function () {
      applyTheme(!isDark(), true);
    });
  }

  // Wire burger toggle (always in DOM)
  wireToggle('themeToggleBurger');

  // Footer toggle is inside a <template> stamped later — wire on first stamp
  // by observing DOM mutations for the footer toggle ID
  var footerObserver = new MutationObserver(function () {
    var footerToggle = document.getElementById('themeToggleFooter');
    if (footerToggle) {
      wireToggle('themeToggleFooter');
      footerObserver.disconnect();
    }
  });
  footerObserver.observe(document.body, { childList: true, subtree: true });

  // React to OS preference change (only when no explicit user choice saved)
  window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', function (e) {
    if (!localStorage.getItem('theme')) {
      applyTheme(e.matches, false);
    }
  });

  // Sync toggle visual state to whatever the no-flash script already set
  syncToggles(isDark());
}

initTheme();
```
