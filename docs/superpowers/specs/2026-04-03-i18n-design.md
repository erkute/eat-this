## Architecture

### New file: `js/i18n.js`

Contains the `TRANSLATIONS` object and the i18n engine. Loaded before `app.js`.

```js
const TRANSLATIONS = {
  en: {
    nav: { ... },
    hero: { tagline, cta },
    start: { sections... },
    musts: { label, subtitle, cards: [...] },
    news: {
      label, title,
      articles: [
        { id, title, excerpt, content, category, categoryLabel, date, dateISO, img, alt }
      ]
    },
    modals: { about, contact, press, impressum, login, agb, datenschutz, cookies },
    footer: { start, news, musts, map, login },
    search: { placeholder, hint, noResults },
    map: { label, title, filterAll, filters: [...] }
  },
  de: { /* same structure, English fallback until translated */ }
}
```

### i18n Engine (in `js/i18n.js`)

```js
let currentLang = localStorage.getItem('lang') || 'en';

function setLang(lang) { ... }        // switches language, saves to localStorage
function applyTranslations() { ... }  // walks DOM, replaces text by key
function t(key) { ... }               // helper: t('hero.tagline') → string
function renderNewsCards() { ... }    // generates news grid HTML from translations
```

### HTML attribute conventions

| Attribute | Usage |
|-----------|-------|
| `data-i18n="key.path"` | Replaces `textContent` |
| `data-i18n-html="key.path"` | Replaces `innerHTML` (modals with HTML content) |
| `data-i18n-placeholder="key.path"` | Replaces `placeholder` on inputs |
| `data-i18n-aria="key.path"` | Replaces `aria-label` |

## Language Switcher

**Placement:** In `.navbar-actions`, left of the search icon.

**Style:** Pill toggle (Variant B — black active state, grey background).

```html
<div class="lang-switcher" id="langSwitcher">
  <button class="lang-btn active" data-lang="en">EN</button>
  <button class="lang-btn" data-lang="de">DE</button>
</div>
```

**Behavior:**
- Clicking a lang button calls `setLang('en')` or `setLang('de')`
- Active class moves to selected button
- Preference saved in `localStorage` key `lang`
- `<html lang="">` attribute updated on switch

---

## News Articles

News articles are **removed from HTML** and **moved into `TRANSLATIONS.en.news.articles`**. A `renderNewsCards(lang)` function generates the card HTML and injects it into `.news-grid`. The existing card structure, modal logic, and `data-*` attributes remain identical — only the source of data changes.

This avoids duplicate HTML and makes articles easy to edit/add per language.

---

## Must-Eat Cards

The 3 must-eat cards stay in HTML (they're structural). Their text attributes (`data-desc`) get language variants:
- `data-en-desc`, `data-de-desc`
- When lang switches, JS updates the active `data-desc` used by the modal

## Code Audit — Issues to Fix

### Security
- `innerHTML` used in news modal content — content comes from `TRANSLATIONS` object (controlled), not user input. Safe. No change needed.
- `three.js` loaded from CDN (`cdnjs.cloudflare.com`) but **never used** → remove script tag (reduces attack surface + saves ~600kb load)

### HTML
- `<html lang="de">` → change to `lang="en"` (matches actual content)
- Missing/wrong `alt` attributes on decorative images → add `alt=""`
- Inline `onclick="openLoginModal()"` on hero button and join button → replace with `data-i18n-action` or keep as-is (low priority, no security issue)
- Duplicate `.navbar-insta::after` rule in CSS (defined twice)

### JavaScript
- `three.js` import reference in app.js (if any) → remove
- Mixed language strings in `app.js` (German notification texts, aria-labels) → move to `TRANSLATIONS`
- Search hint "Tippe um zu suchen..." and "Wonach hast du Appetit?" → move to translations

### CSS — Unused Classes
- Any `three.js` canvas or 3D-related classes
- `.hero-slider` vs `.hero` duplication check
- Full audit pass during implementation
